║
║  AI 辅助 vs AI First
║  ─────────────────────
║  AI 辅助: 在旧有流程(编码/PRD/测试)中引入 AI → 效率提升 10-20%，瓶颈和结构未变
║  AI First: AI 是默认执行者，人类提供方向/判断/约束 → 全流程重构，瓶颈转移后持续突破
║
║  Kanban 的定位: 为 AI Agent 构建工程纪律，让 AI First 从口号变成可操作的 Harness 系统
║  ─────────────────────────────────────────────────────────────────────────────
║  AI First 带来的瓶颈转移（按软件研发周期顺序）:
║
║  需求阶段: 需求进来了没人管
║  任务拆分: 需求拆不成可执行任务——人花几天拆，AI 几分钟写完
║  调度阶段: 任务来了等人工分配和启动——排期、选模板、启动全靠手工
║  开发阶段: 多任务同时跑互相冲突——多个 Agent 改同一文件互相覆盖
║  测试阶段: 做完没人验证质量——AI 两小时写完，人花三天测边界
║  审查阶段: 产出太多审查看不过来——一天部署 3-8 次，靠人看 PR 不现实
║  运维阶段: 上线后问题发现到修复太慢——告警响了没人接，排查靠翻日志
